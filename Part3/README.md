## Part 3

### Environment Configuration

Part 3 builds on the S3PO-GS environment used in Part 2 and adds the dependencies required by Difix3D+.

Enter the Part 3 folder and clone the required repositories:

```bash
cd CV-Project4/Part3
git clone https://github.com/nv-tlabs/Difix3D
git clone https://github.com/nerfstudio-project/gsplat
git clone https://huggingface.co/nvidia/difix
```

The detailed environment configuration should follow the official Difix3D+ repository:

* [Difix3D+](https://github.com/nv-tlabs/Difix3D)

Since Part 3 is implemented on top of the Part 2 S3PO-GS pipeline, first make sure the S3PO-GS environment from Part 2 can run correctly. Then install the additional dependencies required by Difix3D+ and `gsplat`.

### Code Usage

Part 3 runs the BRPO-inspired pseudo-view refinement pipeline with Difix3D+ refinement, reprojection-based fusion, confidence masking, and masked RGB-D optimization.

Enter the BRPO folder:

```bash
cd CV-Project4/BRPO
```

Run the pipeline with one of the provided Part 3 configuration files:

```bash
CUDA_VISIBLE_DEVICES=0 python slam.py --config [part3_config]
```

Available Part 3 configuration files:

```bash
CV-Project4/Part3/configs/dl3dv_2.yaml
CV-Project4/Part3/configs/re10k_1.yaml
CV-Project4/Part3/configs/waymo_405841.yaml
```

For example:

```bash
CUDA_VISIBLE_DEVICES=0 python slam.py --config ../Part3/configs/dl3dv_2.yaml
```

or:

```bash
CUDA_VISIBLE_DEVICES=0 python slam.py --config ../Part3/configs/re10k_1.yaml
```

or:

```bash
CUDA_VISIBLE_DEVICES=0 python slam.py --config ../Part3/configs/waymo_405841.yaml
```

### Notes

Part 3 does not provide additional visualization scripts. The quantitative and qualitative results should be checked from the output folders generated by the corresponding `slam.py` runs.
